# Инструкции для AI-ассистента

## Описание проекта
Этот проект представляет собой интеллектуальную систему для структурирования данных из OCR-документов (паспорта, СНИЛС, водительские удостоверения и др.).
**Ключевая особенность**: Использование RAG (Retrieval-Augmented Generation) на основе **сырых (raw) OCR-текстов** для поиска наиболее релевантных примеров (few-shot), что позволяет системе корректно определять тип документа даже при наличии шума и схожих персональных данных. Структурирование выполняется через LLM (Ollama) с получением валидного JSON.

## Стандарты разработки
Данный проект следует строгим правилам разработки. Любой AI-ассистент, работающий с этим репозиторием, обязан соблюдать следующие стандарты:

## 1. Код и Комментарии
- **Никаких комментариев в коде**: Не оставляйте пояснительные комментарии внутри функций или между строк кода. Код должен быть самодокументированным.
- **Документация на русском**: Все docstrings (описания классов, методов и функций) должны быть написаны на русском языке.

## 2. Работа с Git
- **Атомарные коммиты**: Каждое законченное действие (создание файла, исправление бага, добавление метода) должно быть закоммичено отдельно.
- **Префикс сообщения**: Все сообщения коммитов должны начинаться с `step: ` (например, `step: добавлена логика фильтрации моделей`).

## 3. Тестирование
- **Обязательные тесты**: При добавлении новой логики или изменении существующей, ассистент обязан написать соответствующие тесты (обычно в папке `tests/`).
- **Самопроверка**: Ассистент обязан самостоятельно запустить тесты после внесения изменений и убедиться, что они проходят.

## 4. Качество и Стиль
- Соблюдайте существующую архитектуру проекта.
- Используйте типизацию (Type Hints) в Python коде.
